# DPS utility-file builder

Hand-builds a GM SPS utility file that can be wrapped by DPS's SPAT tool into
a programming archive zip, then run against the simulator.

## Why this exists

DPS's "Programming Archive" zip needs (a) a SPS Interpreter utility file (the
bytecode that drives `$10/$27/$34/...`), (b) zero or more calibration bins,
and (c) a `.tbl` manifest generated by SPAT. The bin files in a captured
`spsDebug` zip are gzipped DOTE-encoded payloads, not utility files - DPS
rejects them. There is no free public download of a utility file we can
reuse, so we synthesise one.

## Reference

Wire format reverse-engineered from the open-source CCRT3700 C# implementation
of the GM SPS Interpreter at
<https://github.com/MikeMcNamara-Ascential/CCRTCommon3700>. The
`reference/` directory holds copies of the relevant files:

- `Interpreter.cs` - 100-byte PTI header + 24-byte SPS header parser
- `OpCodeHandler.cs` - Interpreter 3 (GMLAN) op-code handlers
- `UtilityFileDisplay.cs` - the official viewer GUI's read path
- `OpCodeTestProgram.cs` - per-opcode test vectors (concrete byte literals)
- `Interpreters_September_01_2009.docx` - the GM Interpreters spec
- `OpCodeAssessment.xls` - GM op-code coverage analysis

Cross-checked against the local `DPS Programmers Reference Manual.pdf`
(Appendix D, Interpreter 3 / GMLAN op-codes table).

## Wrap into a DPS archive (SPAT inside dps.exe)

1. Open `C:\DPS\dps.exe`.
2. From the main window, click **SPAT** (Service Programming Archive Tool).
3. Click **Create a New Archive**.
4. Archive name: anything <= 32 chars, no extension (e.g. `gmsim_demo01`).
5. Template: pick **`calfil00.DTM`** (utility-only - matches our zero-cal demo)
   or **`calfil01.DTM`** (utility + 1 cal - use if you also want to drop in
   a payload bin).
6. For the **Utility File** row: click **Select File** ->
   `tools\dps_utility_builder\dist\utility_demo.bin`.
7. (If `calfil01.DTM`) for the cal row: select any small bin -
   `dist\dummy_cal.bin` is provided.
8. Optionally tick **Disable Auto File Conversion** in DPS options before
   building, so SPAT does not try to convert our already-binary input to
   another format.
9. Click **Build**. SPAT generates the `.tbl` and zips everything.

## Run against the simulator

1. Start `GmEcuSimulator.exe`.
2. In DPS: set Vehicle Architecture = GMLAN, Protocol = Class 2 / Global A.
   Set the J2534 device to `GM ECU Simulator`.
3. Click **Program**, point at the archive zip from step 9 above.
4. Watch the simulator's log -- you should see `$10 02`, `$27 01`,
   then teardown.

## Extending

`build_utility_file.build_minimal_demo` is intentionally small. To add
op-codes (e.g. `$34 REQUEST_DOWNLOAD` to actually flash a cal), edit that
function and add `Instruction(op_code=..., action=..., gotos=...)` lines.
`OpCodeHandler.cs` is the canonical reference for action / goto field
semantics per op-code.
